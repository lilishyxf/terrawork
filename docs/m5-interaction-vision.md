# M5 交互升级愿景:拖拽部署 + 瓦片场景(草案,未实现)

> 用户拍板的产品方向,等瓦片素材到位后实现。决策:**甲(你组队、向导分工)**。
> 这是 TerraWorks 区别于 Pixel Agents 那类"单向镜子"的具象化——拖拽**真的驱动编排**,不只是动画。

## 核心交互:拖拽部署(deploy / recall)
- **院子 = 待命区**:全部专家 NPC 默认在院子,**黑白(grayscale)休眠**——一眼看出"谁待命"。
- **拖进工作区 = 部署上岗**:NPC 上色、进入可被分派状态;**你定阵容**。
- **拖回院子 = 召回休眠**:重新变黑白。
- **拖动时举手**:复用现有 talk 帧(按住/拖拽 → 招手)。

## 编排关系:甲 = 你组队、向导分工
- 向导(M2.7 自动匹配)**只在你已部署的队员里**分派任务;未部署的不参与。
- 即:**用户定"有哪些人可用",向导定"谁干哪张卡"**——掌控 + 智能兼得。
- (备选乙=纯手动指派、丙=偏好,已弃。)

## 落到事件模型(合 §1 红线#1 + 复用 M4 写半边)
- 拖拽 = 玩家交互 → 落 `user_interact{target:<npc>, interaction:"deploy"|"recall"}` 事件(schema 已预留 user_interact)。
- 编排器消费:维护"已部署专家集合";`_decide` 分派 builder 时**只在已部署集合内**选/实例化(扩展 M2.7 的匹配:候选受部署集约束)。
- 投影:NPC 增 `deployed` 维度 → View 决定上色/黑白。
- 走 `POST /sessions/{sid}/interact`(M4 写端点的延伸;当前 v0.1 只做了 /command、/hitl)。

## 依赖:瓦片场景(先决条件)
- 拖拽需要**清晰的 drop 区**:工作区房间(规划/实现/验证/评审)= drop 目标,院子 = 待命/休眠区。
- 所以 **瓦片场景 + 拖拽部署一起做**(房间布局即 drop 区定义);用户正在找瓦片素材(CC0/原创/朋友定制)。

## 实现要点(可行性已评估,均不难)
- 黑白休眠:Phaser tint / grayscale pipeline。
- 拖拽:Phaser `setDraggable` + drop 区命中检测(用瓦片房间矩形)。
- 拖时举手:现有 `_talk` 动画。
- 真正的活在**事件映射 + 编排器部署集约束**(后端)。

## 实现顺序(瓦片到位后)
1. 瓦片场景渲染(房间/地面/墙替换矩形框)+ 定义 drop 区。
2. user_interact 写端点 + 编排器"已部署集"约束分派(甲)。
3. 前端拖拽 + 黑白休眠 + drag 举手 + deploy/recall 发事件。
4. 投影加 deployed 维度。
